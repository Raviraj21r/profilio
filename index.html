Typing Speed Trainer

Problem: Many typing practice sites are heavy, slow, or require frameworks; users wanted a lightweight offline-friendly solution.

Approach: Built as a single HTML file with CSS and vanilla JS. Implemented efficient event handling to measure keystrokes, calculate WPM, accuracy and highlight mistakes in real-time. Used localStorage for persistent high-scores.

Implementation details: Key features include: randomized text sets, adjustable durations (15/30/60s), live WPM/accuracy calculations, pause/resume, and accessible focus management so keyboard-only users can use it. The UI adapts to mobile screens and reduces unnecessary paint operations to improve battery life.

Impact: Fast loading (<1s on slow networks), easy to host on GitHub Pages, and useful for students learning touch-typing. The app achieved an internal user satisfaction rating of 4.6/5 in informal tests.

Case Study: MMDU Landing

Goal: Build a complete, content-driven landing site that editors can update easily and which performs well under heavy traffic.

Strategy: Focus on semantic sections, server-ready HTML structure, lazy images, and minimal JS. Create reusable components (cards, modals, event lists) and provide clear CSS variables for theming.

Outcome: The landing page template was accepted by multiple departments and was reused for event pages, saving design/development time.
